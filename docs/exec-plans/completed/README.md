# Completed Exec Plans

用于沉淀已完成计划的收口文档，避免执行证据长期留在 `active/`。

## 进入 completed 的标准

1. 对应计划的开发、测试、验收结论已明确。
2. 关键证据（命令、报告、偏差与结论）可追溯。
3. active 文档中存在指向 completed 收口文档的链接或标记。

## 收口文档最小内容

1. 目标与范围
2. 执行结果与证据摘要
3. 偏差与遗留风险
4. 后续行动（若有）

## 约束

- completed 文档仅追加，不覆盖历史事实。
- 若发现结论变化，新增补充收口文档，不篡改原记录。
